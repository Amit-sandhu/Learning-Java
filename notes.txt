Identifiers : the names given to define the variables, functions, classes, enum, anything.

The general rules for naming variables are :
    Names can contain letters, digits, underscores, and dollar signs
    Names must begin with a letter
    Names should start with a lowercase letter, and cannot contain whitespace
    Names can also begin with $ and _
    Names are case-sensitive ("myVar" and "myvar" are different variables)
    Reserved words (like Java keywords, such as int or boolean) cannot be used as names

java devs use camelcase convention for identifiers. (eg : camelCase, firstNum, thisIsNewIdentifier)
All constants identifiers are writen in caps. 

VARIABLES :

    Variables are containers for storing data values.

    In Java, there are different types of variables, for example:

    String - stores text, such as "Hello". String values are surrounded by double quotes
    int - stores integers (whole numbers), without decimals, such as 123 or -123
    float - stores floating point numbers, with decimals, such as 19.99 or -19.99
    char - stores single characters, such as 'a' or 'B'. Char values are surrounded by single quotes
    boolean - stores values with two states: true or false

there are two types of variables : primitive and non primitive:
primitive are base and predefined default data types by the language/ smallest unit of storage.(eg : int, boolean, double)
non-primitive are the derived datatypes. ( eg classes, arrays)

==============================
JAVA MEMORY (Easy Explanation)
==============================

There are 3 main memory areas in Java:

1. STACK
- Stores method calls.
- Stores local variables.
- Stores references (addresses) to objects.
- Fast memory.
- Memory is automatically removed when a method finishes.

Example:
Student s = new Student();

Here, 's' is stored in the Stack.

--------------------------------------

2. HEAP
- Stores all objects created using 'new'.
- Stores instance (non-static) variables.
- Objects remain here until no references point to them.
- Garbage Collector removes unused objects.

Example:
Student s = new Student();

The actual Student object is stored in the Heap.

--------------------------------------

3. METHOD AREA (METASPACE)
- Stores class information.
- Stores method definitions.
- Stores constructor information.
- Stores static variables.

Example:
static String company = "Google";

'company' is stored once in the Method Area and shared by all objects.

--------------------------------------

Example:

class Student{
    static String company = "Google";
    String name;
    int age;
}

Student s = new Student();

Method Area:
- Student class
- company

Stack:
- reference variable 's'

Heap:
- Student object
- name
- age

--------------------------------------

Easy Analogy:

Method Area = University rulebook
(Defines what a Student is and stores shared static data.)

Heap = Hostel
(The actual students/objects live here.)

Stack = Reception Desk
(Keeps a list of room numbers (references) and temporary work.)

When you write:
Student s = new Student();

1. JVM reads the Student blueprint from the Method Area.
2. Creates a Student object in the Heap.
3. Stores the reference to that object in variable 's' on the Stack.

Remember:

Objects live in the HEAP.
References live in the STACK.
Static variables live in the METHOD AREA.

Think of a university.

Heap is like the hostel. Every student (object) actually lives there. 
When you write new Student(), you're building a new hostel room and putting a student in it.

Stack is like the reception desk. It doesn't contain the students—it only contains a 
list of room numbers (references) and temporary information needed while work is being done.

Method Area (Metaspace) is like the university rulebook. It contains the blueprint of what 
a Student is (its fields, methods, constructors) and shared information like static variables.
